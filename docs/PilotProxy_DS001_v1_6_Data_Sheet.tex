\section*{Introduction}
PilotProxy is a standalone CUDA F-statistic detector for ATSC 1.0 DTV pilot tones.  It estimates and detects sub-noise-floor DTV data-shelf signals by measuring pilot excess above a local reference floor.  The package includes a CUDA kernel, public C API, Python wrappers, a precomputed ATSC reference weight bank, and a GNU Radio based validation testbench.

The project is intentionally packaged as a self-contained DTV pilot detector and testbench.  It does not require external receiver data products, instrument-specific metadata, or deployment-specific masking workflows.

ATSC A/53 Part 2 defines the RF/transmission subsystem for ATSC digital television and the VSB transmission characteristics used by 8-VSB.  The package uses the ATSC pilot relationship, expressed here as the positive offset \(\mathrm{pilot\_below\_data\_db}=11.3\dB\), to convert a one-bin pilot-excess measurement to an equivalent DTV data-shelf SNR \cite{atsc_a53_part2_2011}.

\subsection*{Features}
\begin{itemize}
  \item Standalone CUDA detector for sub-noise-floor ATSC 1.0 pilot detection.
  \item Fixed 128-sample detector rows using packed complex 4+4 bit input.
  \item Three matched-filter terms: target pilot bin, lower reference bin, upper reference bin.
  \item Exact \code{uint64} target/reference power readback.
  \item Detection by rational fixed-point threshold comparison; no division in the mask path.
  \item Public threshold coordinate is DTV shelf SNR in dB.
  \item Kernel threshold coordinate is equivalent fixed-point F-statistic half-threshold.
  \item GNU Radio ATSC 1.0 waveform generation, audit, AWGN injection, quantization, and SNR evaluation.
  \item Physical-channel helpers and shipped weights for U.S. UHF channels 14-36.
\end{itemize}

\section{General Description}

PilotProxy estimates a weak ATSC 1.0 signal that is below the broadband noise floor.  The DTV data shelf is spread across the nominal 6 MHz channel, while the ATSC pilot is narrow and can be measured with a matched detector.  The detector measures the pilot excess in one fine bin and converts that measurement to an equivalent 6 MHz data-shelf SNR.

The CUDA kernel is intentionally narrower than the physical interpretation.  It receives a row-major packed detector matrix, a 3-term packed weight vector, and a rational half-threshold.  It returns accumulated powers and a mask.  It does not know about ATSC, dB, shelf SNR, or physical channels.

\section{Signal Definitions}

\subsection{Raw F-Statistic}

For one evaluated frame, the kernel accumulates:
\begin{equation}
  P_t,\qquad P_{r1},\qquad P_{r2}.
\end{equation}
The target term \(P_t\) is centered on the ATSC pilot.  The two reference terms estimate the local non-pilot floor.  The deployed raw F-statistic is:
\begin{equation}
  \boxed{F=\frac{2P_t}{P_{r1}+P_{r2}}.}
\end{equation}
The production \code{NumDen} API writes:
\begin{equation}
  \code{num}=P_t,\qquad \code{den}=P_{r1}+P_{r2}.
\end{equation}
Therefore host code must use \(F=2\,\code{num}/\code{den}\), not \(\code{num}/\code{den}\).

\subsection{Pilot Excess and PNR}

The raw F-statistic level is:
\begin{equation}
  R_F=10\log_{10}F.
\end{equation}
This is not the physical pilot-to-noise ratio, because \(F=1\) means no pilot excess.  The operational pilot-excess ratio is:
\begin{equation}
  \boxed{\rho=F-1.}
\end{equation}
The one-bin pilot-excess PNR is:
\begin{equation}
  \boxed{\mathrm{PNR}_{\mathrm{bin},dB}=10\log_{10}(F-1).}
\end{equation}
If \(F\leq 1\), the dB-valued pilot-excess PNR is undefined and JSON reports encode it as \code{null}.

\subsection{DTV Shelf SNR}

The DTV data-shelf SNR is the ATSC data/sync shelf PSD relative to the non-DTV noise PSD.  The pilot is excluded from the data-shelf numerator.  The standard-mode conversion is:
\begin{equation}
  \boxed{\mathrm{SNR}_{\mathrm{shelf},dB}=
  \mathrm{PNR}_{\mathrm{bin},dB}
  -10\log_{10}\left(\frac{B_{\mathrm{DTV}}}{B_{\mathrm{bin}}}\right)
  +\mathrm{pilot\_below\_data\_db}
  -10\log_{10}(\eta_p).}
\end{equation}
The default constants are:
\begin{equation}
 B_{\mathrm{DTV}}=6\,\mathrm{MHz},\quad
 B_{\mathrm{bin}}=3051.7578125\,\mathrm{Hz},\quad
 \mathrm{pilot\_below\_data\_db}=11.3\dB,
 \quad \eta_p=1.0.
\end{equation}
With these constants, the spreading loss is:
\begin{equation}
  10\log_{10}\left(\frac{6\times 10^6}{3051.7578125}\right)=32.936\dB.
\end{equation}

\section{Threshold Interpretation}

The public detection threshold is \code{threshold_snr_shelf_db}.  Host software converts it to the F-statistic coordinate and then to the rational half-threshold used by the kernel.

\begin{align}
\mathrm{PNR}_{\mathrm{bin,thr},dB} &= S_{\mathrm{shelf,thr},dB} - \mathrm{pilot\_below\_data\_db} + 10\log_{10}(B_{\mathrm{DTV}}/B_{\mathrm{bin}})+10\log_{10}(\eta_p),\\
\rho_{\mathrm{thr}} &= 10^{\mathrm{PNR}_{\mathrm{bin,thr},dB}/10},\\
F_{\mathrm{thr}} &= 1+\rho_{\mathrm{thr}},\\
T_{\mathrm{half}} &= \frac{F_{\mathrm{thr}}}{2}=\frac{N_{\mathrm{half}}}{D_{\mathrm{half}}}.
\end{align}
The CUDA kernel evaluates:
\begin{equation}
  P_t D_{\mathrm{half}} > N_{\mathrm{half}}(P_{r1}+P_{r2}).
\end{equation}
Exact equality is not a positive detection; this matches the
CHIME positive-excess convention and the deployed rational-half API.


\notebox{For a threshold of \(-26\dB\) shelf SNR with the included reference geometry, the package reports approximately \(\mathrm{PNR}_{\mathrm{bin}}=-4.364\dB\), \(F_{\mathrm{thr}}=1.36610123\), and \(T_{\mathrm{half}}\approx0.68305\).}

The shipped weight bank covers physical channels 14 through 36.  The default target is physical channel 14.  U.S. channel 14 occupies 470-476 MHz, so the ATSC pilot is placed at 470.309441 MHz.  The U.S. television channel table assigns 6 MHz bands and lists channel 14 as 470-476 MHz \cite{ecfr_73603}.

\section{Receiver Integration Contract}

The CUDA detector core is receiver-independent.  Receiver integrations supply metadata and arrays above the kernel boundary:
\begin{itemize}
  \item \code{DetectorCoreProfile}: locked kernel contract, including \(K=128\), \(N=3\), int4 packed input, uint64 powers, and rational half-threshold input.
  \item \code{ReceiverProfile}: RF band, channelizer geometry, spectral sense, frame size, number of input streams, quantization defaults, fine-bin ENBW, and pilot-capture efficiency.
  \item \code{stream_map}: stream-index metadata for feed, polarization, beam, or other receiver paths.
  \item Runtime commands such as \code{detect}, \code{evaluate-snr}, and \code{chime-run} bind those metadata files to concrete input arrays and output products.
\end{itemize}

The kernel still receives only packed rows, packed weights, uint64 powers, and a rational half-threshold.  For one selected DTV channel:
\begin{equation}
N_{\mathrm{rows}} = N_{\mathrm{input\ streams}}\,
\frac{\mathrm{frame\_size\_samples}}{128}.
\end{equation}
The default combine mode is an incoherent power sum over streams.  Powers are summed before the statistic is formed:
\begin{equation}
F = \frac{2\sum_r P_{t,r}}{\sum_r P_{r1,r}+\sum_r P_{r2,r}},
\end{equation}
where \(r\) indexes detector rows.  The public result-schema name for this rule is \code{all_rows_summed_before_ratio}.  Per-stream diagnostic packing is available for integration debugging, but the public detection mode combines streams before forming \(F\).

\section{Result Schema and Mask Convention}

Every public JSON output includes a versioned \field{result_schema} object with \code{schema_version=pilot_proxy_result_v2}.  The schema records statistic definitions, layout metadata, calibration constants, threshold conversion fields, the fixed-point contract, and the mask convention.

The fixed-point contract records:
\begin{itemize}
  \item \field{detector_window_samples=128};
  \item \field{reference_offset_bins=2} nominally, with \field{skipped_guard_bins=1} and adaptive placement metadata copied from the weight manifest;
  \item \field{packed_complex_bits=8};
  \item \field{sample_bits_per_component=4};
  \item \field{power_accumulator=uint64}.
\end{itemize}

The generic masking convention is exclusion masking:
\begin{equation}
\bar{P}_{\mathrm{after}} =
\frac{\sum_b P_b(1-M_b)}{\sum_b (1-M_b)}.
\end{equation}
Here \(M_b=1\) means that block/sample is masked and excluded.  Masked samples are not zero-filled before averaging.

\section{Reference PFB and Detector Resolution}

The standalone testbench uses a neutral reference channelizer:
\begin{itemize}
  \item real ADC sample rate: 800 MS/s;
  \item modeled band: 400 MHz bandwidth;
  \item PFB: 4 taps, 2048-point real input;
  \item usable positive-frequency channels: 1024;
  \item coarse-channel spacing: \(800\,\mathrm{MHz}/2048=390.625\,\mathrm{kHz}\);
  \item fine detector-bin spacing: \(390.625\,\mathrm{kHz}/128=3051.7578125\,\mathrm{Hz}\).
\end{itemize}
The fine detector-bin width is the bandwidth term used in the default SNR conversion.

\section{Data Format}

Each sample is one signed \code{int8_t} value containing a packed complex int4 sample:
\begin{equation}
  \text{bits }[7:4]=\mathrm{real},\qquad \text{bits }[3:0]=\mathrm{imag}.
\end{equation}
Each nibble is decoded as two's-complement signed int4 with range \([-8,7]\).  The generated weight bank uses the symmetric component range \([-7,7]\) to avoid the negative-minimum edge case in generated detector ROMs.

Input matrices are row-major:
\begin{equation}
  \mathrm{d\_in}[m,k]=\mathrm{d\_in}[mK+k],\qquad K=128.
\end{equation}
For batched input:
\begin{equation}
  \mathrm{d\_in}[b,m,k]=\mathrm{d\_in}[(bM+m)K+k].
\end{equation}

\section{Fixed-Point Sizing}

The locked 128-sample window is selected by the int4 dot-product growth bound.  One complex-product component is a sum of two int4-by-int4 products and is conservatively bounded by 128 per tap.  Accumulating 128 taps gives an absolute real/imaginary dot-product bound of 16384, which requires signed 16-bit storage.  A 256-sample power-of-two window would exceed the signed-int16 target.  The fixed-point design metadata also records that the magnitude-squared bound fits in \code{uint32} and the full block power sum fits in \code{uint64}.

\section{Limitations}

\begin{itemize}
  \item Installed wheels and sdists carry the weight bank and \code{configs/} resources; the CUDA kernel is not packaged and builds from the cloned source tree.
  \item The CUDA API is default-stream only.
  \item The shipped weight bank covers physical channels 14-36 only.
  \item Standalone validation is GNU Radio ATSC synthetic validation only.  The CHIME/baseband real-data workflow ships separately in the \code{pilot_proxy.chime} subsystem and is specified by its runbooks, not by this data sheet.
  \item Receiver profiles are an integration contract, not a validated telescope pipeline.
  \item No operational threshold is derived from real telescope null statistics; the CHIME workflow instead uses the parameter-free norm-corrected mask ($F>\mu_0$).
  \item The kernel detects pilot excess.  It does not decode ATSC transport streams.
  \item Calibration constants are explicit.  Standard mode uses the ATSC 11.3 dB relationship; calibrated mode should record any override values.
\end{itemize}
